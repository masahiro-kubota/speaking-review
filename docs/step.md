# 英会話振り返りツール 開発手順

## 目的

DMM英会話などのレッスン音声を使って、

* 文字起こし
* 2話者分離
* 自分の発話抽出
* 英語添削フィードバック生成

を行うMVPを段階的に開発する。

---

## 開発方針

* いきなり全部作らない
* まずは動く最小構成を作る
* 精度改善や自動化は後回しにする
* 各ステップで「使い物になるか」を確認しながら進める

---

## Step 1. 開発環境を準備する

### 目的

音声処理とLLM呼び出しを試せる最小環境を作る。

### 作業

* Ubuntu に Python 3.10 以上を用意する
* `ffmpeg` をインストールする
* Python仮想環境を作成する
* 必要なライブラリを入れる

### 想定ライブラリ

* faster-whisper または WhisperX
* pyannote 系依存ライブラリ
* requests / openai などのAPIクライアント
* pydantic や PyYAML などの設定補助

### 完了条件

* 音声ファイルを読み込める
* Pythonスクリプトを実行できる

---

## Step 2. 文字起こしだけ実装する

### 目的

まずは音声からテキストを取得できるようにする。

### 作業

* 音声ファイルを1つ入力する
* 英語で文字起こしする
* 結果をJSONまたはテキストで保存する

### 出力例

```json
[
  {
    "start": 0.0,
    "end": 2.3,
    "text": "Hello, how are you?"
  }
]
```

### 確認ポイント

* 音声が問題なく処理できるか
* レッスン音声である程度読める精度か
* 処理時間が許容範囲か

### 完了条件

* 1本の音声から transcript が生成される

---

## Step 3. 話者分離を追加する

### 目的

文字起こし結果に `Speaker A / Speaker B` を付ける。

### 作業

* 話者分離を有効にする
* 2人会話前提で設定する
* 出力に話者ラベルを付与する

### 出力例

```json
[
  {
    "speaker": "Speaker A",
    "start": 0.0,
    "end": 2.3,
    "text": "Hello, how are you?"
  },
  {
    "speaker": "Speaker B",
    "start": 2.5,
    "end": 4.0,
    "text": "I'm good, thank you."
  }
]
```

### 確認ポイント

* 2人に概ね分けられるか
* 長めの発話でラベルが安定しているか
* 被り発話や短い相槌で大きく崩れないか

### 完了条件

* transcript に話者ラベルが付く

---

## Step 4. 英語添削プロンプトを接続する

### 目的

各発話に対して、英語教師としての添削結果を生成する。

### 作業

* 1文ずつLLMに渡す
* 指定フォーマットで結果を返す
* 元文と添削結果を対応づけて保存する

### プロンプト方針

* corrected sentence を最初に出す
* natural alternatives を2〜3件出す
* explanation は短くする
* spoken English を重視する

### 確認ポイント

* 1文単位で安定して返るか
* 説明が長すぎないか
* 会話向けの自然な表現が出るか

### 完了条件

* 各発話に対して添削結果が返る

---

## Step 5. Markdownレポートを生成する

### 目的

後で見返しやすい形式にまとめる。

### 作業

* セッション単位でMarkdownを出力する
* 元の文、修正文、言い換え、説明をまとめる
* 必要なら時刻や話者情報も残す

### 出力例

```markdown
# 英会話振り返り

## Sentence 1
Original:
I go to there yesterday.

Corrected Sentence:
I went there yesterday.

Natural Alternatives:
1. I went there yesterday.
2. I was there yesterday.
3. I went over there yesterday.

Explanation:
過去形と不要な "to" を修正。
```

### 確認ポイント

* 読み返しやすいか
* 1レッスン1ファイルで管理できるか

### 完了条件

* review.md が自動生成される

---

## Step 6. CLIとしてまとめる

### 目的

毎回バラバラにスクリプトを叩かず、1コマンドで処理できるようにする。

### 作業

* `main.py` か `cli.py` を作る
* 入力音声パスを引数で受ける
* 出力先ディレクトリを指定できるようにする

### 実行イメージ

```bash
python main.py --input lesson.mp3 --output output/
```

### 完了条件

* 1コマンドで最後まで処理できる

---

## Step 7. 実データで評価する

### 目的

MVPが本当に役立つかを確認する。

### 作業

* 5〜10本程度の実レッスン音声で試す
* 文字起こし精度をざっくり確認する
* 話者分離のズレを確認する
* 添削結果が学習に役立つかを見る

### 見るポイント

* 明らかな誤認識が多すぎないか
* Speaker A/B が途中で頻繁に逆転しないか
* 添削の内容が冗長すぎないか
* 本当に復習したくなる出力か

### 完了条件

* 継続利用したいと思える品質に達する

---

## Step 8. 必要なら改善する

### 改善候補

* 文字起こしモデルの変更
* APIベースASRへの切り替え
* 話者分離パラメータ調整
* プロンプト改善
* 似たミスの集約
* レッスン単位の要約追加

### 優先順位

1. 文字起こし精度
2. 話者分離の安定性
3. レポートの読みやすさ
4. 自動化やUI

---

## MVP完了の定義

以下が満たされればMVP完成とする。

* 音声ファイルを入力できる
* 英語の文字起こしができる
* 2話者に分けられる
* 自分の発話だけ抽出できる
* 各発話に添削を付けられる
* Markdownの振り返りレポートを出力できる

---

## 推奨する進め方

最初の1週間は、以下だけで十分。

1. 音声 → 文字起こし
2. 文字起こし → 2話者ラベル
3. 自分の発話だけ抽出
4. 3〜5文だけLLM添削
5. Markdown出力

これで価値が出れば、その後に改善する。
